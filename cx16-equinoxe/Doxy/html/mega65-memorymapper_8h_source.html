26 // Remap some of the eight 8K memory blocks in the 64K address space of the 6502 to point somewhere else in the first 1MB memory space of the MEGA65.
27 // After the remapping the CPU will access the mapped memory whenever it uses instructions that access a remapped block.
28 // See section 2.3.4 in http://www.zimmers.net/cbmpics/cbm/c65/c65manual.txt for a description of the CPU memory remapper of the C65.
29 // remapBlocks: Indicates which 8K blocks of the 6502 address space to remap. Each bit represents one 8K block
30 // - bit 0 Memory block $0000-$1fff. Use constant MEMORYBLOCK_0000.
31 // - bit 1 Memory block $2000-$3fff. Use constant MEMORYBLOCK_2000.
32 // - bit 2 Memory block $4000-$5fff. Use constant MEMORYBLOCK_4000.
33 // - bit 3 Memory block $6000-$7fff. Use constant MEMORYBLOCK_6000.
34 // - bit 4 Memory block $8000-$9fff. Use constant MEMORYBLOCK_8000.
35 // - bit 5 Memory block $a000-$bfff. Use constant MEMORYBLOCK_A000.
36 // - bit 6 Memory block $c000-$dfff. Use constant MEMORYBLOCK_C000.
37 // - bit 7 Memory block $e000-$ffff. Use constant MEMORYBLOCK_E000.
38 // lowerPageOffset: Offset that will be added to any remapped blocks in the lower 32K of memory (block 0-3).
39 // The offset is a page offset (meaning it is multiplied by 0x100). Only the lower 12bits of the passed value is used.
40 // - If block 0 ($0000-$1fff) is remapped it will point to lowerPageOffset*$100.
41 // - If block 1 ($2000-$3fff) is remapped it will point to lowerPageOffset*$100 + $2000.
