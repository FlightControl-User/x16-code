1 // A flexible sprite multiplexer routine for 32 sprites.
2 // Usage:
3 // - Once:
4 // - plexInit(screen): Initialize the data structures and set the screen address
5 // Each frame:
6 // - Set x-pos, y-pos and pointer in PLEX_XPOS[id], PLEX_YPOS[id], PLEX_PTR[id]
7 // - plexSort() Sorts the sprites according to y-positions and prepares for showing them. This uses an insertion sort that is quite fast when the relative order of the sprites does not change very much.
8 // - plexShowSprite() Shows the next sprite by copying values from PLEX_XXX[] to an actual sprite. Actual sprites are used round-robin. This should be called once for each of the 24 virtual sprites.
9 // - plexFreeNextYpos() Returns the Y-position where the next sprite is available to be shown (ie. the next pos where the next sprite is no longer in use showing something else).
10 // - plexShowNextYpos() Returns the Y-position of the next sprite to show.
11 //
12 // In practice a good method is to wait until the raster is beyond plexFreeNextYpos() and then call plexShowSprite(). Repeat until all 32 sprites have been shown.

43 // Uses an insertion sort:
44 // 1. Moves a marker (m) from the start to end of the array. Every time the marker moves forward all elements before the marker are sorted correctly.
45 // 2a. If the next element after the marker is larger that the current element
46 // the marker can be moved forwards (as the sorting is correct).
47 // 2b. If the next element after the marker is smaller than the current element:
48 // elements before the marker are shifted right one at a time until encountering one smaller than the current one.
49 // It is then inserted at the spot. Now the marker can move forward.
