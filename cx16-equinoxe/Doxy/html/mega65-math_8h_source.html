1 // MEGA65 CPU MATHS ACCELERATION REGISTERS
2 // Every MEGA65 contains a combined 32-bit hardware multiplier and divider. This device
3 // takes two 32-bit inputs, MULTINA and MULTINB, and simultaneously calculates:
4 // • the 64-bit product of MULTINA and MULTINB
5 // • the 32-bit whole part of MULTINA / MULTINB
6 // • the 32-bit fractional part of MULTINA / MULTINB
7 // It is always updating the outputs based on the inputs, so there is no need to take special
8 // action when changing the inputs. The multiplier takes 1 cycle to calculate, and the updated result will thus be available immediately.
9 // The hardware divider, however, can take up to 16 cycles depending on the particular inputs. Thus programmers should insert a short
10 // delay after changing the inputs before reading the output. As this delay is so short, it can be implemented by simply reading the first
11 // byte of the result four times consecutively, as the 4th read will occur after the result has settled.
